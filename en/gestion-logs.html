Introduction

The log management functionality allows users to view, filter, and generate reports on system activities. This tool is essential for tracking actions, conducting audits, and maintaining a history of events.

Log Management Interface

The Log Management window contains the following main elements:

Interface Elements

  1. Filter Combobox
    • User: Filters logs by a specific user.
    • Log: Filters by a specific type of recorded action.
    • Date: Filters logs by a specific date.
  2. Log Table:
    • Displays activity records in a table format with the following columns:
      • User: The name of the user who pemformed the action.
      • Date: The date and time of the event.
      • Action: A description of the recorded activity.
  3. Action Buttons:
    • Create file: Generates a file with the current records for export or saving.
    • Update data: Reloads the log table with the most recent data.
  4. Title:
    • A header indicating that you are in the Log Management section.

Usage Flow

Viewing Logs

When the window opens, the log table displays recent activity records.

The table includes details such as the user's name, the action date, and a brief event description.

Filtering Logs

You can use the dropdown filters to narrow the displayed logs:

  • By User: Select a username from the first dropdown to view only actions performed by that user.
  • By Log Type: Use the second dropdown to filter by a specific type of action (e.g., Login or User Creation).
  • By Date: Filter the records by selecting a date in the third dropdown.

Creating a File

  1. Clic the Create file button.
  2. A file containing the currently visible record in the table wil be generated.
  3. This file can be used as a backup or to share the event history.

Refresing Data

  1. Click the Update data button to reload the logs in the table.
  2. This action is useful if new activities have been recorded in the system while you are using the window.

Common Errors and Solutions

Filters do not show results:

  • Cause: Filters are configured but lack logic to modify the table data.
  • Solution: This behavior will be improved in future updates

File is not generated when clicking "Create file" button:

  • Cause: The method associated with the button does not yet include file generation logic.
  • Solution: Contact the developer to implement this functionality.

Recommendations

  • Use the filters to narrow the range of records and simplify analysis.
  • Refresh the data before generating a file to ensure the most recent records are included.
