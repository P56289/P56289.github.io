Interface Description

The message-sending window includes the following elements:

  1. Recipient field: Allows you to enter the recipient's email address.
  2. Subject field: Allows you to enter the subject of the message.
  3. Message field: Text area for writing the body of the message.
  4. Attachment selection: Allows you to attach a file up to 10 MB.
  5. Send button: Sends the email.
  6. Back button: Returns to the program's main window.

Attachment Selection

To attach a file:

  1. Click on the Select button to open the file explorer.
  2. Select the file you want to attach.
  3. The name of the selected file will appear in the corresponding field.

Note: The attachment size must not exceed 10 MB.

Email Sending

To send an email:

  1. Enter the recipient's email address in the Recipient field.
  2. Enter the subject of the message in the Subject field.
  3. Write the message in the Message area.
  4. If necessary, attach a file by clicking on the Select button.
  5. Click on the Send button.

"Back" Button

The Back button allows returning to the program's main window.

  1. Clicking this button closes the message-sending window.
  2. The main window automatically opens.

Usage Example

A typical usage example is described below:

  1. Enter user_destination@gmail.com in the Recipient field.
  2. Enter Message Test in the Subject field.
  3. Write This is a test message in the Message area.
  4. (Optional) Attach a small file by clicking Select.
  5. Click on the Send button.
  6. Check the confirmation message: Email sent successfully.
  7. Click on the Back button to return to the main window.

Troubleshooting

Common Problems and Their Solutions

  1. Error: Invalid email format.

    2. Verify that the entered email address is valid.

  2. Error: The attachment exceeds 10 MB.

    3. Select a smaller file.

  3. Error: SMTP server error.

    4. Ensure your internet connection is active and that the account used has SMTP access enabled.

  4. Unexpected error: Check the error details and contact the system administrator if the issue persists.
