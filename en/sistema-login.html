Introduction

The login system is designed to ensure that only authorized users can access the application. This manual details how to use the login interface, as well as possible actions in case of errors.

Login interface

The login window includes the following elements:

  • User field: Allows you to enter the registered user name.
  • Password field: Allows you to enter the password corresponding to the user.
  • Login button: Starts the credentials validation process.
  • Error message: Appears in red if the credentials entered are incorrect.

User instructions

How to log in

To log in, the user must follow the instructions below.

  1. Open the app.
  2. In the Username field, enter your username.
  3. In the Password field, enter your password.
  4. Click the Login button.

If the credentials are correct, a success message will appear and you will access the next screen of the application.

But if they are incorrect, a red error message will be displayed stating Incorrect credentials, please try again.

Maximum number of failed attempts

The system allows a maximum of 5 login attempts. If this limit is reached:

  • A pop-up message will appear stating Too many failed attempts. The application will close.
  • The application will automatically close to protect security.

Error handling

Incorrect credentials

The message Incorrect credentials, please try again indicates that the credentials you entered do not match any registered users. Please verify your username and password.

Too many attempts

If you enter incorrect credentials 5 times in a row, the system will block access and the application will close.

Recommendations

  • Strong Passwords: Make sure you use a strong password that combines uppercase and lowercase letters, numbers, and special characters.
  • Password Recovery: If you do not remember your credentials, contact your system administrator to reset them.
  • Safe shutdown: To close the login window without attempting to log in, use standard operating system functions to close the application.

Example login

Successful login

  1. The user enters his/her credentials.
    Username: 'admin'
    Password: 'password123'
  2. The system validates the credentials and displays Login successful.
  3. The main system window opens.

Faild login

  1. The user enters his/her credentials.
    Username: 'admin'
    Password: 'incorrect'
  2. The system detects that the credentials are incorrect and displays Incorrect credentials, please try again.
  3. If this happens 5 times in a row, the application closes automatically.

Additional notes

  • Privacy: The password is not visible while typing, as the field is protected with a masked input format.
  • Security: All passwords are securely verified using a validation system (they are not stored in plain text).
  • Close protection: If the app closes due to too many failed attempts, you will need to restart to try again.
